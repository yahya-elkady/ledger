Subscriptions

A subscription bills a customer on a plan's schedule, with optional trials and immediate or end-of-period cancellation. Subscriptions are processed by Stripe.

API key write scope mode-isolated

Create a subscription

POST/v1/subscriptions
Idempotency-Key required
FieldTypeNotes
customer_idstringrequiredAn existing customer's id.
plan_idstringrequiredAn existing plan in the same mode — otherwise 404.
payment_method_idstringrequiredProcessor-tokenized payment method.
trial_endtimestampoptionalRFC 3339, in the future — free until then.
metadataobjectoptionalAny JSON object.

Returns 201 with the subscription (status and period dates come from the processor). Errors: 400 validation_error, 404 (unknown plan), 502 processor_error.

curl -X POST http://localhost:8080/v1/subscriptions \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "customer_id": "1f3c…",
    "plan_id": "7e9a…",
    "payment_method_id": "pm_card_visa",
    "trial_end": "2026-07-01T00:00:00Z"
  }'

List subscriptions

GET/v1/subscriptions
Query paramNotes
statusoptionalFilter, e.g. active, trialing, canceled.
limitoptionalDefault 20, max 100.
cursoroptionalFrom the previous next_cursor.

Returns 200 with {"data": [...], "next_cursor": "…"}.

Retrieve a subscription

GET/v1/subscriptions/{id}

Returns 200 or 404 not_found (also for cross-mode access).

Cancel a subscription

POST/v1/subscriptions/{id}/cancel
Idempotency-Key required
FieldTypeNotes
at_period_endbooloptionalfalse (default): cancel now — status becomes canceled, canceled_at is set. true: keep serving until current_period_end, then cancel (status stays as-is until the period closes; the body may be omitted entirely for an immediate cancel).

Returns 200 with the updated subscription. Errors: 404, 502 processor_error.

# cancel at the end of the current billing period
curl -X POST http://localhost:8080/v1/subscriptions/5c2e…/cancel \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{"at_period_end": true}'

Webhooks: when the processor finalizes lifecycle changes (e.g. an at-period-end cancellation completing, or a payment issue), you receive subscription.canceled / subscription.updated events.

Not yet available

Changing an existing subscription's plan or payment method (PUT /v1/subscriptions/{id}) is planned but not implemented — cancel and re-create instead.
