Bank Accounts

A bank account is a payout destination — where your money goes when you create a payout. Only an opaque processor account id plus display metadata is stored; never routing or account numbers.

API key admin scope

The bank account object

FieldTypeNotes
idstringUnique identifier.
merchant_idstringThe owning merchant.
processorstringstripe or plaid — which rail pays out to it.
processor_acct_idstringThe processor's opaque account token.
last4stringDisplay only.
bank_namestringDisplay only.
currencystringThe account's currency, if set.
is_defaultboolMarks your preferred payout destination.
created_attimestampRFC 3339 UTC.

Register a bank account

POST/v1/bank-accounts
Idempotency-Key required
FieldTypeNotes
processorstringrequiredstripe or plaid.
processor_acct_idstringrequiredToken from the processor's bank-linking flow (e.g. Stripe external account, Plaid Link).
last4stringoptionalFor display in your UI.
bank_namestringoptionalFor display in your UI.
currencystringoptionalIf sent, must be a supported ISO 4217 code.
is_defaultbooloptionalDefaults to false.

Returns 201 with the bank account object.

curl -X POST http://localhost:8080/v1/bank-accounts \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "processor": "stripe",
    "processor_acct_id": "ba_1NXgZq…",
    "last4": "6789",
    "bank_name": "First Example Bank",
    "currency": "USD",
    "is_default": true
  }'

List bank accounts

GET/v1/bank-accounts

Returns 200 with {"data": [bank_account, …]} — your full list (no pagination).

Delete a bank account

DELETE/v1/bank-accounts/{id}

Soft-deletes the payout destination; past payouts that reference it remain intact.

Returns 204, or 404 not_found.

Why admin scope?

Bank accounts and payouts are where money leaves the platform — the most sensitive operations a key can perform. Keep admin-scoped keys on tightly controlled servers, separate from the write keys your checkout path uses.
