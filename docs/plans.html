Plans

A plan is a recurring price — "$15.00 every month" — that customers subscribe to. Creating a plan also registers it with the processor (plans are Stripe-backed: a Product + recurring Price pair).

API key write scope mode-isolated

Create a plan

POST/v1/plans
Idempotency-Key required
FieldTypeNotes
namestringrequired
amountintrequiredPositive integer, minor units.
currencystringrequiredSupported ISO 4217 code.
intervalstringrequiredOne of day, week, month, year.
interval_countintoptionalDefaults to 1.

Returns 201 with the plan. Errors: 400 validation_error, 502 processor_error (registration with Stripe failed — nothing was saved; retry with the same idempotency key).

curl -X POST http://localhost:8080/v1/plans \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "name": "Pro Monthly",
    "amount": 1500,
    "currency": "USD",
    "interval": "month"
  }'

List plans

GET/v1/plans

Returns 200 with {"data": [plan, …]} — all your active plans for the current mode (no pagination; deleted plans are excluded).

Delete a plan

DELETE/v1/plans/{id}

Soft-deletes the plan so no new subscriptions can be created on it. Existing subscriptions keep billing until they are canceled.

Returns 204, or 404 not_found.

Plans are immutable

There is no update endpoint — pricing changes are a new plan. Create "Pro Monthly v2", subscribe new customers to it, and migrate or grandfather existing subscribers as your policy dictates.
